| Campo | Valor |

| **Verificado em** | 2026-06-24 · Ginga telemidia/ginga (C++) · Ubuntu 22.04 |

## 1. Resumo

Este documento (`ncl id="ncluaEx"`) demonstra a **integração de um objeto imperativo NCLua** (`media` com `src` apontando para um script `.lua`) dentro de um documento NCL declarativo. O exemplo reaproveita toda a estrutura interativa do app "Primeiro João" (vídeo principal segmentado, menu de ritmos musicais, formulário, settings global) e acrescenta o nó `changes` (`../script/counter.lua`), que **conta quantas vezes o usuário trocou de ritmo** e, ao final, **desenha o resultado na tela** usando a API gráfica `canvas` do NCLua. O foco do RFC é mostrar como a lógica Lua troca eventos com o documento NCL via propriedades de atribuição (`property`) e âncoras de apresentação (`area`).

## 2. Conceitos NCL demonstrados

- **Objeto imperativo NCLua**: `media` com `src="../script/counter.lua"` integrada ao restante do documento declarativo.
- **Comunicação NCL ↔ Lua por atribuição**: `property name="add"` no nó Lua, escrita pelo NCL via `bind role="set" interface="add"` e lida pelo `handler1` (evento `ncl/attribution/start`).
- **Comunicação NCL ↔ Lua por apresentação**: `area id="print" label="fim"`, disparada pelo NCL via `start` e tratada pelo `handler2` (evento `ncl/presentation/start`).
- **API gráfica do NCLua** (`canvas`): `attrSize`, `attrColor`, `drawRect`, `attrFont`, `drawText`, `flush`.
- **API de eventos do NCLua** (`event`): `event.register`, `event.post` (devolve evento ao NCL com `action='stop'`).
- **Reúso de nós** com `refer` / `instance="instSame"` (vídeo `animation`, settings `globalVar`, nó Lua `changes`).
- **Conectores causais importados** via `importBase` (`causalConnBase.ncl`).
- **Switch declarativo** com `ruleBase` (seleção de música por `service.currentFocus`) e com `bindRule`/`defaultComponent` (formulário PT/EN por `system.language`).
- **Nó settings** (`application/x-ginga-settings`) com variáveis globais `service.interactivity` e `service.currentFocus`.
- **Segmentação temporal** de vídeo por `area` com `begin`/`end`, transições (`transitionBase`) e navegação por foco (`focusIndex`/`moveRight`/`moveLeft`).

## 3. Estrutura do documento

### 3.1 Layout — regiões e descritores

A `regionBase` define uma região raiz `backgroundReg` (100% × 100%, `zIndex=1`) que aninha todas as demais:

| Região | left / top | width × height | zIndex |
|--------|-----------|----------------|--------|
| `backgroundReg` | 0 / 0 | 100% × 100% | 1 |
| `screenReg` | 0 / 0 | 100% × 88% | 2 |
| `frameReg` | 5% / 6.7% | 18.5% × 18.5% | 3 |
| `iconReg` | 87.5% / 11.7% | 8.45% × 6.7% | 3 |
| `shoesReg` | 15% / 60% | 25% × 25% | 3 |
| `formReg` | 57.25% / 9.83% | 37.75% × 70.2% | 3 |
| `intReg` | 92.5% / 91.7% | 5.07% × 6.51% | 3 |
| `chorinhoReg` | 2.5% / 91.7% | 11.7% × 6.51% | 3 |
| `rockReg` | 25% / 91.7% | 11.7% × 6.51% | 3 |
| `technoReg` | 47.5% / 91.7% | 11.7% × 6.51% | 3 |
| `cartoonReg` | 70% / 91.7% | 11.7% × 6.51% | 3 |
| `changesReg` | 0 / 90% | 100% × 10% | 4 |

`changesReg` é a região do objeto Lua: uma faixa inferior de tela cheia (`zIndex=4`, acima de tudo) onde o `canvas` desenha o retângulo preto e o texto amarelo do resultado.

Descritores relevantes (`descriptorBase`):

- `backgroundDesc` → `backgroundReg`; `screenDesc` → `screenReg` (vídeo principal).
- `photoDesc` → `frameReg`, `explicitDur="5s"`, `descriptorParam transparency=0.6`.
- `dribleDesc` → `frameReg`, com `transIn="trans1"` (fade 2s) e `transOut="trans2"` (barWipe 1s).
- `iconDesc` → `iconReg`, `explicitDur="6s"`.
- `formDesc` → `formReg`, `focusIndex="1"`, `explicitDur="15s"`.
- Botões de ritmo com navegação por foco: `chorinhoDesc` (`focusIndex=2`), `rockDesc` (`focusIndex=3`), `technoDesc` (`focusIndex=4`), `cartoonDesc` (`focusIndex=5`), encadeados por `moveRight`/`moveLeft`.
- `intDesc` → `intReg` (ícone liga/desliga interatividade).
- `audioDesc` (sem região — áudio do chorinho).
- **`changesDesc` → `changesReg`**: descritor do objeto NCLua.

### 3.2 Conectores

Os conectores são **importados** de `causalConnBase.ncl` via `<importBase documentURI="causalConnBase.ncl" alias="conEx"/>`. Os efetivamente usados neste documento:

| Conector | Condição → Ação | Parâmetros |
|----------|-----------------|------------|
| `onBeginStart` | `onBegin` → `start` (par, unbounded) | — |
| `onBeginStart_delay` | `onBegin` → `start` com `delay="$delay"` | `delay` |
| `onEndStop` | `onEnd` → `stop` | — |
| `onEndStart` | `onEnd` → `start` | — |
| `onBeginSet_varStart` | `onBegin` → seq(`set $var`, `start`) | `var` |
| `onBeginSet_var` | `onBegin` → `set $var` | `var` |
| `onEndSet_var` | `onEnd` → `set $var` | `var` |
| `onBeginVarStart` | (`onBegin` **and** `var == true`) → `start` | (assessment) |
| `onBeginStartSet_var_delay_duration` | `onBegin` → seq(`start`, `set $var` delay/duration) | `var`, `delay`, `duration` |
| `onKeySelectionStopSet_varStart` | `onSelection key=$keyCode` → seq(`stop`, `set $var`, `start`) | `var`, `keyCode` |
| `onSelectionSet_varStop` | `onSelection` → seq(`set $var`, `stop`) | `var` |
| `onSelection_orSet_varStopStart` | `onSelection` (or, unbounded) → seq(`set $var`, `stop`, `start`) | `var` |

`onBeginVarStart` usa `compoundCondition operator="and"` combinando `onBegin` com um `assessmentStatement` (`attributeAssessment role="var"` do tipo `attribution` comparado a `true`): só dispara se a interatividade estiver ligada.

### 3.3 Mídias

Mídias do corpo (raiz e dentro dos `context`):

| id | src / tipo | descritor | observações |
|----|-----------|-----------|-------------|
| `background` | `../media/background.png` | `backgroundDesc` | fundo |
| `animation` | `../media/animGar.mp4` | `screenDesc` | vídeo principal segmentado |
| `drible` | `../media/drible.mp4` | `dribleDesc` | clipe com transições |
| `photo` | `../media/photo.png` | `photoDesc` | `property name="top"` (animação de posição) |
| `newChanges` | `refer="changes"` (`instSame`) | — | reúso do nó Lua na raiz |
| **`changes`** | **`../script/counter.lua`** | **`changesDesc`** | **objeto NCLua** |

As `area` (âncoras) do vídeo `animation` definem a temporização global:

- `segDrible` (`begin=12s`), `segPhoto` (`begin=41s`), `segIcon` (`begin=45s` `end=51s`), `segLua` (`end=61s`), `segCred` (`end=64s`).

O **nó Lua `changes`** expõe duas interfaces que são a ponte com o NCL:

- `<area id="print" label="fim"/>` — âncora de **apresentação** (presentation); quando o NCL a inicia, o `handler2` do script é acionado.
- `<property name="add"/>` — propriedade de **atribuição** (attribution); cada `set add=1` vindo do NCL chega ao `handler1`.

No `context interactivity`, o nó `globalVar` (`application/x-ginga-settings`) declara `service.interactivity` (`value="true"`) e `service.currentFocus`. No `context menu`, o `switch musics` seleciona `rock`/`techno`/`cartoon` (`.mp4`) por regras sobre `service.currentFocus`; o `switch form` (em `advert`) escolhe `ptForm.htm`/`enForm.htm` por `system.language`.

### 3.4 Elos e temporização

Fluxo temporal do documento (elos da raiz):

1. **`port entry` → `animation`**: o vídeo principal inicia o documento.
2. **`lMusic`** (`onBeginStart_delay`): `onBegin animation` → `start background` e `start menu`, ambos com `delay=5s`. Após 5s entram o fundo e o menu de ritmos.
3. **`lDrible`** (`onBeginStart`): `onBegin animation.segDrible` (12s) → `start drible`.
4. **`lPhoto`** (`onBeginStartSet_var_delay_duration`): `onBegin animation.segPhoto` (41s) → `start photo` + `set photo.top=290` com `delay=1s`, `duration=3s` (foto desliza).
5. **`lEnd`** (`onEndStop`): `onEnd animation` → `stop background` e `stop interactivity`.
6. Elo (sem id, `onEndStop`): `onEnd animation.segCred` (64s) → `stop menu`.

**Integração com o NCLua** (núcleo deste RFC):

- **Incremento do contador (NCL → Lua, atribuição).** No `context menu`:
  - `lChoro` (`onSelectionSet_varStop`): ao selecionar `imgChorinho` → `set choro.soundLevel=1`, **`set changes.add=1`**, `stop musics`.
  - `lOthers` (`onSelection_orSet_varStopStart`): ao selecionar `imgRock`/`imgTechno`/`imgCartoon` → `set choro.soundLevel=0`, **`set changes.add=1`**, `stop musics`, `start musics`.

  Cada `set changes.add=1` gera no script o evento `ncl/attribution/start name='add'`, capturado por `handler1`, que faz `counter = counter + evt.value` e devolve ao NCL `event.post{ ... action='stop' }` (encerrando a atribuição). Ou seja, **cada troca de ritmo soma 1 ao contador mantido em Lua**.

- **Exibição do resultado (NCL → Lua, apresentação).** Elo (sem id, `onEndStart`):
  - `onEnd animation.segLua` (61s) → **`start newChanges.print`** (âncora `fim` do nó Lua, via reúso `newChanges`).

  Isso dispara `handler2` (`ncl/presentation/start label='fim'`), que pinta o `canvas` de preto, escreve em amarelo (fonte `vera 24 bold`) `"O número de vezes que você trocou de ritmo foi: "..counter`, faz `canvas:flush()` e responde com `event.post{ ... action='stop' }`.

- **Camada de interatividade** (contexto `interactivity`): `lInt` liga a interatividade quando o vídeo começa (`onBeginSet_varStart` setando `service.interactivity=true` e mostrando `intOn`); `lOn`/`lOff` alternam liga/desliga pela tecla `INFO` (`onKeySelectionStopSet_varStart`).

- **Camada de anúncio** (contexto `advert`): `lIcon` (`onBeginVarStart`) mostra o `icon` em `segIcon` **somente se** a interatividade estiver ativa; `lBegingShoes` (tecla `RED`) abre `shoes` + `form` e reduz o vídeo via `set reusedAnimation.bounds`; `lFormFocus`/`lEndForm` ajustam foco e restauram o vídeo (`bounds=0,0,222.22%,222.22%`).

## 4. Execução

```bash
cd "Primeiro joao/PrimeiroJoao/PrimeiroJoao/Exemplos"
ginga 11nclua.ncl
```

**Comportamento esperado:** o vídeo principal (`animGar.mp4`) começa em tela cheia. Após 5s surgem o fundo e o menu inferior com os botões de ritmo (chorinho/rock/techno/cartoon); ao longo do vídeo aparecem o clipe de drible (12s, com fade/barWipe), a foto deslizante (41s) e o ícone de interação (45–51s). A cada vez que o usuário seleciona um botão de ritmo, o NCL envia `set add=1` ao script Lua, que incrementa um contador interno (e a música do `switch musics` é trocada). Aos 61s (`segLua`), o NCL inicia a âncora `print` do nó Lua e o script desenha na faixa inferior, em texto amarelo sobre fundo preto, a frase "O número de vezes que você trocou de ritmo foi: N". Aos 64s o menu para e ao fim do vídeo o documento é encerrado.

**Resultado verificado:** ✅ O documento carrega e roda no Ginga atual; o nó NCLua `counter.lua` é integrado corretamente, recebe as atribuições `add` a cada troca de ritmo e renderiza o texto-resultado via `canvas` ao final do vídeo. — ver captura [`../screenshots/11nclua.png`](../screenshots/11nclua.png).

## 5. Observações

- **Dependências de mídia local.** O exemplo usa caminhos relativos `../media/` e `../script/`; é necessário executar a partir do diretório `Exemplos/` para que as referências resolvam. Mídias presentes e confirmadas no diretório `media/`: `background.png`, `animGar.mp4` (~38 MB), `drible.mp4`, `photo.png`, `icon.png`, `intOn.png`, `intOff.png`, `shoes.mp4`, `chorinho.png`, `rock.png`/`techno.png`/`cartoon.png`, `choro.mp4`, `rock.mp4`/`techno.mp4`/`cartoon.mp4`, `ptForm.htm`/`enForm.htm`.
- **Conector importado.** `causalConnBase.ncl` precisa estar no mesmo diretório `Exemplos/` (está presente); o `alias="conEx"` é usado em todos os `xconnector` do documento.
- **Script NCLua.** `../script/counter.lua` registra dois handlers: `handler1` (atribuição da propriedade `add`) e `handler2` (apresentação da âncora `fim`). O contador (`local counter`) é estado em memória do objeto Lua; o reúso `newChanges` (`refer="changes" instance="instSame"`) garante que a mesma instância (e portanto o mesmo `counter`) seja usada ao disparar a âncora `print`.
- **Particularidades / limitações.**
  - O texto exibido depende da fonte `vera` estar disponível no ambiente do Ginga; caso contrário o `attrFont` pode usar fonte substituta.
  - A regra `int` (`service.interactivity`) está comentada na `ruleBase`; a interatividade é controlada pelos elos `lInt`/`lOn`/`lOff` e pelo conector `onBeginVarStart`.
  - O vídeo `animGar.mp4` é grande (~38 MB); se o repositório usar **Git LFS**, garantir que os ponteiros tenham sido resolvidos (`git lfs pull`) antes de executar, caso contrário as mídias `.mp4` não serão reproduzidas.
  - Encoding do `.ncl`: `ISO-8859-1` (atenção a acentuação ao editar).
